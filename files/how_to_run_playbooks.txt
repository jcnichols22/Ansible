
How to Run Each Ansible Playbook
================================

Assuming you are in the root of your Ansible project. Your ansible.cfg is already set up, so you do not need to specify the inventory file each time.

---

onboard_all.yml
   ansible-playbook playbooks/onboard_all.yml

bootstrap_lxc_ssh_keys.yml (run once when LXC root SSH key auth is missing)
   ansible-playbook playbooks/bootstrap_lxc_ssh_keys.yml -K
   # Injects ~/.ssh/ansible.pub into root authorized_keys for LXC guests via Proxmox pct exec.

onboard_lxc.yml
   ansible-playbook playbooks/onboard_lxc.yml -K
   # Cleans known_hosts, bootstraps root SSH keys via Proxmox, remediates stale APT repos.

update_docker_media.yml
   ansible-playbook playbooks/update_docker_media.yml
   # Updates and restarts all Docker media services on the media server.

remediate_lxc_apt_repos.yml
   ansible-playbook playbooks/remediate_lxc_apt_repos.yml
   # Manual cleanup of stale APT repos on LXC containers.

update_lxc_containers.yml
   ansible-playbook playbooks/update_lxc_containers.yml
   # Updates all LXC containers, includes stale-repo remediation.

update_servers.yml
   ansible-playbook playbooks/update_servers.yml
   # Updates all Ubuntu servers (proxmox_nodes, media_server).

sync_proxmox_inventory.yml
   ansible-playbook playbooks/sync_proxmox_inventory.yml -K
   # Discovers LXC/VM guests on Proxmox nodes, refreshes inventory.

sync_uptime_kuma.yml
   ansible-playbook playbooks/sync_uptime_kuma.yml -K
   # Syncs service monitors to Uptime Kuma.

sync_service_hosts_docker.yml
   ansible-playbook playbooks/sync_service_hosts_docker.yml -K
   # Discovers Docker services with published ports on the media server and refreshes a managed docker_services inventory block.

openclaw_onboard_machine.yml
   ansible-playbook playbooks/openclaw_onboard_machine.yml -l <target_host> -K
   # Onboards a single machine to openclaw automation.

onboard_proxmox_hosts.yml
   ansible-playbook playbooks/onboard_proxmox_hosts.yml -l proxmox_nodes
   # Onboards Proxmox hosts with base users and SSH keys.

---

If you use a different inventory or need extra variables, adjust the command accordingly.
